Урок: Использование JAR-файлов: Основы

JAR-файлы упакованы в формат файлов ZIP, так что вы можете использовать их для "ZIP-подобных" задач, таких как сжатие данных без потерь, архивация, развертывание и распаковка архива. Эти задачи относятся к наиболее общему использованию JAR-файлов, вы можете реализовать многие преимущества JAR-файлов, используя только эти основные возможности.

Даже если вы хотите получить преимущества от более сложной функциональности, обеспечиваемой форматом JAR-файла, такой как электронная подпись, вам сначала нужно познакомиться с базовыми операциями.

Чтобы выполнять базовые задачи с JAR-файлами, вы используете утилиту архивирования JavaTM Archive, предоставляемую как часть Java Development Kit. Поскольку утилита Java Archive вызывается командой jar, мы для краткости будем называть ее "утилитой Jar".

Как конспект некоторых тем, которые будут рассмотрены в данном уроке, в следующей таблице дана сводка общих операций над JAR-файлами:

ОперацияКоманда
Создание JAR-файла jar cf jar-файл входной файл(ы)
Просмотр содержания JAR-файла jar tf jar-файл
Выделение содержимого из JAR-файла jar xf jar-файл
Выделение заданных файлов из JAR-файла jar xf jar-файл архивированный-файл(ы)
Выполнение приложения, пакетированного как JAR-файл
(версия 1.1)		 
jre -cp приложение.jar MainClass
Выполнение аплета, пакетированного как JAR-файл
(версия 1.2 - требуется заголовок манифеста Main-Class)

Вызов аплета, пакетированного как JAR-файл

Создание JAR-файла

Этот раздел показывает вам, как использовать утилиту Jar для пакетирования файлов и каталогов в JAR-файл.

Просмотр содержания JAR-файла

Вы можете вывести содержание JAR-файла, чтобы увидеть, что в нем содержится, без распаковки JAR-файла.

Выделение содержимого из JAR-файла

Вы можете использовать утилиту Jar для распаковки JAR-файла. При выделении файлов утилита Jar делает копии требуемых файлов и записывает их в текущий каталог, воспроизводя ту структуру каталогов, которая была в архиве.
