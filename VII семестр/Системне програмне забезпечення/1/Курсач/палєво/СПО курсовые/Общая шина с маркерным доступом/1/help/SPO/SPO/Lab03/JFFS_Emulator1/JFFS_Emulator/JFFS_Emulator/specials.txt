Размер блока и страницы задаются параметрами. 
Флэш-память поддерживает однократное программирование страницы. (подумать об использовании для служебных полей)
Директория - 1. Плоское файловое пространство.
Команды:
format path pageSize pageAmount(создает и размечает требуемую карту флеш-памяти)
mount
unmount
blockstat(id)
create fileName
unlink fileName
list
open fileName
close fileName
read fd offset size
write fd offset data

link fileName newFileName
truncate fileName size.

В памяти хранятся записи об изменениях в самой системе, а не файлы.
Каждая запись интерпретируется драйвером. В результате чего получаем представление файловой системы в 1 файле.
По умолчанию - только читать. Регистрировать наличие изменений, при тру - запросить сохранение данных при анмаунте.
Вести счетчик записей страницы и на странице.
При записи составить карту актуальности записей. Перезаписывать страницу, на которой меньше всего актуальных записей. 
	Если есть несколько - записывавшуюся наименьшее кол-во раз.
	
	Архитектура:
	1. Список соответствия имен и дескрипторов файлов.
	2. Список блоков памяти с изменениями.
	3. Карта перекрытия файлового пространства.
	4. Управление (интерпретатор команд)
	5. супер-менеджер управления памятью
